agentforce-md

Convert Claude Code markdown to Salesforce Agentforce Agent Script

agentforce-md lets you define a Salesforce Agentforce AI agent using the same markdown files Claude Code already reads — CLAUDE.md for agent config and .claude/agents/*.md for topics and actions. A single slash command converts those files into a deployable .agent bundle and pushes it to your org.

No hand-editing XML. No switching tools. Write markdown, run one command, get a live agent.

How it works

  1. 1
    Write your agent in markdown

    Describe agent behaviour in CLAUDE.md. Add sub-agents under .claude/agents/*.md. Use YAML frontmatter to declare variables, action bindings, and routing guards.

    your-project/
  CLAUDE.md                ← agent config + variables
  .claude/
    agents/
      order-support.md     ← topic: actions + routing
      general-faq.md
  2. 2
    Run /agentforce-convert in Claude Code

    The skill reads your markdown, resolves variable bindings, and generates a complete .agent bundle in the agents/ directory. Use /agentforce-discover first to check which Flows and Apex classes already exist in your org.

  3. 3
    Deploy to your Salesforce org

    The skill calls sf agent deploy to push the bundle. If referenced Flows or Apex classes are missing, use /agentforce-scaffold to generate stubs first.

Install

curl -sSL {{ base_url }}/install.sh | bash

Installs to ~/.claude/agentforce-md/ and registers skills in Claude Code. To update later: python3 ~/.claude/agentforce-md-install.py --update

Prerequisites

  • Python 3.10+ — python3 --version
  • Claude Code installed — ~/.claude/ must exist — install guide ↗
  • Salesforce CLI — optional, required for deploy — sf --version

Skills

/agentforce-convert

Parse your project’s markdown files and generate a deployable Agent Script bundle. Optionally deploy to your org in the same step.

start here

/agentforce-discover

Inspect a Salesforce org and report which Flows, Apex classes, and Retrievers your SKILL.md files reference — showing what’s deployed vs. what’s missing.

audit org first

/agentforce-scaffold

Generate stub Flow XML and Apex class files for targets that don’t yet exist in the org, so you have a starting point to build from.

create missing stubs

/agentforce-run

Execute a single SKILL.md action against a live org via REST API — useful for testing individual actions without going through the full agent.

test in isolation

Recommended workflow

Write CLAUDE.md + agents/*.md
→
Discover /agentforce-discover
→
Scaffold /agentforce-scaffold
→
Convert /agentforce-convert

Usage after install

# Open Claude Code in any Salesforce project and type:
/agentforce-discover          # see what's in your org
/agentforce-scaffold          # generate missing Flow / Apex stubs
/agentforce-convert           # convert markdown → .agent → deploy

# Or call the CLI directly (no Claude Code required):
~/.claude/agentforce-md/bin/agentforce-md convert \
  --project-root . --agent-name MyAgent --org MyOrg
